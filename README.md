# Example: Two Sum Problem

# Approach 1: Brute Force O(nÂ²)
def two_sum_brute(nums, target):
    for i in range(len(nums)):
        for j in range(i+1, len(nums)):
            if nums[i] + nums[j] == target:
                return [i, j]
    return []

# Approach 2: Hash Map O(n)
def two_sum_optimal(nums, target):
    seen = {}
    for i, num in enumerate(nums):
        complement = target - num
        if complement in seen:
            return [seen[complement], i]
        seen[num] = i
    return []
```

### 2. **Complexity Analysis**

Every solution includes detailed Time and Space complexity:

```
Time Complexity:  O(n log n)  - Sorting algorithm
Space Complexity: O(n)        - For the output array

Why this complexity?
- Sorting takes O(n log n) due to comparison operations
- We store all n elements in output, hence O(n) space
- Can't do better than O(n log n) for comparison-based sort

Trade-offs:
- Could use counting sort for O(n) if range is known
- Space could be O(1) with in-place sort (modifies input)
```

```

---

### Key Interview Tips

1. **Clarify the Problem**
   - Ask about edge cases
   - Confirm constraints
   - Ask about expected output format

2. **Discuss Approach**
   - Explain brute force first
   - Then optimize
   - Discuss trade-offs

3. **Code Cleanly**
   - Use meaningful variable names
   - Add comments for complex logic
   - Handle edge cases

4. **Test Thoroughly**
   - Test with provided examples
   - Test edge cases (empty, single, large)
   - Test with negative/special values

5. **Optimize**
   - Improve time complexity
   - Reduce space complexity
   - Explain why improvements work

6. **Communicate**
   - Think out loud
   - Explain your reasoning
   - Ask for feedback
